Love Link - Project Context (Mar 10, 2026)
=========================================

Purpose & Product Vision
------------------------
Love Link is a single-page, installable PWA (HTML/CSS/JS only) for long-distance couples. It provides a private, shared space with realtime sync, partner presence, and push notifications. The UI is intentionally “Apple-like” with a liquid glass aesthetic, but performance on low-end phones is critical.

Current Status Summary
----------------------
- Functional: miss signals, mood sync, notes, goals, dates, reminders, letters, game, wallpapers, admin panel, push.
- Visual: iOS-style glass UI, now moving toward a light/white Apple aesthetic with colored accents.
- Performance: background motion removed; performance mode exists; remaining glass blur kept but reduced for mobile.
- Deployment: static on GitHub Pages, service worker caching with version bump to force refresh.

Core Features (Current)
-----------------------
1. Miss Signal
   - Large circular "I MISS YOU" button sends realtime event + optional push.
   - Shows overlay + haptic + sound on receiver.

2. Mood Sync
   - Single slider (0-100) with mood labels + ring glow.
   - Synced across devices.

3. Relationship Streak + Daily Note
   - Days together based on anniversary date.
   - Daily note saved and synced.

4. Our Songs (Spotify embed)
   - Playlist iframe + open link.

5. Goals + Important Dates (with calendar grid)
   - CRUD goals + dates; calendar highlights date entries.

6. Reminders v2
   - Reminder creation with time + repeat (once/daily/weekdays/weekends).
   - Done / Skip / Snooze (+10m), streaks, done/skip counts.
   - Stats header (your done, partner done, best streak).

7. Open-When Letters v2
   - Title/body, optional photo attachment (base64).
   - Unlock by mood or by datetime.
   - Open receipts with overlay view (no alert popups).

8. Mini Game (Ping Pong)
   - Canvas game with increasing difficulty + scores synced.

9. Shared Wallpaper
   - Upload photo, set blur + dim, apply to both devices.

10. Admin Super Panel
   - Pair code generation, admin code updates, theme override.
   - Realtime connect, diagnostics, logs.
   - Replay events, view sync diff, export/import JSON.
   - Feature flags: Reminders/Letters/Game/Wall.
   - Hard reset with double confirm + optional cloud wipe.

Architecture & Files
--------------------
Single static app. No build system.
- index.html: UI layout and screens.
- styles.css: Apple-like glass UI, animations, mobile styling.
- app.js: All logic, state, Firebase sync, push, UI events.
- sw.js: PWA cache + push notifications.
- manifest.webmanifest + icons/: PWA metadata/icons.

Runtime Environment
-------------------
- Runs entirely in the browser (PWA).
- iOS: Add-to-Home-Screen limitations (push works in modern iOS, but with OS limits).
- Android: Chrome PWA expected; best behavior on recent devices.

Realtime & Push
---------------
Firebase Realtime Database + Anonymous Auth.
Events + shared state:
- rooms/{pairCode}/state: full synced state.
- rooms/{pairCode}/events: event stream (miss, updates, deletes).
- rooms/{pairCode}/presence, profiles: online + partner name.
Push:
- FCM VAPID key embedded in app.js (FCM_VAPID_KEY).
- Tokens stored in roomTokens/{room}/{uid}/{token}.
- sw.js handles push -> showNotification.

State Model (app.js defaultState)
---------------------------------
- auth: loggedIn, name, role, pairCode, partnerName
- settings: anniversary, adminCode, themeOverride, notifications, featureFlags
- preferences: focusMode, haptics, soundCue, perfMode, blur, motion, radius, scale
- wallpaper: imageData, blur, dim
- sync: firebaseConfig, enabled, connected, lastEventAt, lastNotifiedMissAt
- push: token
- songs, goals, dates, reminders, letters, letterTombstones
- mood, dailyNote, missLog, pulse
- admin logs + filter
- game state + metrics

UI & Theme Design Notes
-----------------------
- Visual direction: Apple-like “liquid glass,” soft gradients, clean typography.
- Themes: Guy (blue accent) and Girl (pink accent), both now intended on a light/white base.
- Glass: uses `backdrop-filter` on cards/overlays; mobile uses reduced blur to avoid artifacts.
- Animations: some subtle CSS animations remain (breathe, gleam) but heavy background motion removed.

Recent Performance Work (Mar 2026)
----------------------------------
- Removed tilt/axis background motion (disabled motion loop).
- Disabled animated background layer (bg-layer) to reduce GPU load.
- Simplified wallpaper transforms (static scale).
- Added Performance Mode (settings) that reduces blur, shadows, motion.
- Reduced reminders timer to 30s.
- Added drag/velocity drawer pull with friction (iOS feel).
- Fixed mobile extra scroll band by adjusting shell padding and overflow.
- Cache version bumps to prevent stale GitHub Pages assets (APP_VERSION).

Known UX Issues / Risks
-----------------------
- On some Android devices, `backdrop-filter` causes gray/boxy artifacts. We reduced it on mobile, but keep it for the iOS glass look.
- Heavy assets (wallpaper and letter images) are stored as base64 in Realtime DB, which can cause lag and quota usage.
- Full state sync is large and frequent; could be optimized by storing diffs and moving images to Storage.

Key UX/Behavior Notes
---------------------
- Drawer: drag handle or drawer itself, friction + velocity snapping.
- Letters: delete uses tombstones to avoid reappearing on sync.
- Miss signals: only show if event is fresh and not already notified.
- Feature flags hide tabs and auto-return to Home if disabled.
- When wallpaper is present, bg-layer is hidden.

Files You’ll Touch Most
-----------------------
1) app.js
   - UI bindings, state, Firebase, push, game loop, reminders/letters logic.
   - PWA cache refresh logic in setupPWA/forceRefreshPWA.
2) styles.css
   - Glass UI, animations, mobile layout.
   - Performance Mode styles: body.perf.
3) index.html
   - All sections/screens, admin panel, overlays.
4) sw.js
   - Cache name/version, push handling.

Known Constraints / Potential Issues
------------------------------------
- Heavy payloads: wallpaper + letter images are base64 in RTDB (quota risk).
- Full state publish still includes large objects.
- Some animations remain (breathe/gleam). If phone still lags, reduce further.
- Notifications on iOS behave differently due to PWA limitations.

If You Need to Optimize Further
-------------------------------
1) Move images to Firebase Storage and store URLs in RTDB.
2) Remove remaining blur/backdrop-filter on low-end devices.
3) Throttle CSS animations (or set prefers-reduced-motion).
4) Limit max base64 image size in compressImageFile.
5) Reduce full-state sync frequency or switch to event-only sync.
6) Reduce heavy box-shadows on mobile; avoid nested glass layers.
